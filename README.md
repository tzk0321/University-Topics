# 🍳 C2C 模式的食物共享網頁平台 (Food-Sharing)
這是一個為大學專題開發的平台，所以主題源自於對大學校園生活的觀察。身為大學生，我們常發現校內活動（如講座、會議）結束後，往往剩下大量的餐盒與便當，讓主辦方深感困擾。與此同時，對於預算有限的學生族群來說，這些剩餘資源其實是生活中的「小確幸」。

因此，我希望建立一個「互利共享」的平台，透過資訊透明化解決物資浪費問題，同時強化校園社群的資源連結。<br><br>

## 核心功能
* **即時發布與管理：** 提供直觀介面，使用者可快速發布食物資訊（地點、期限），實現資訊即時上傳與分享。
* **模糊關鍵字搜尋 ：** 使用者可依據類別和關鍵字等，高效定位目標食物，大幅提高媒合效率。
* **使用者互動機制：** 具備留言與通知功能，確保分享者與取用者之間能夠進行有效且即時的溝通。<br><br>

## 使用技術
* **Frontend:** HTML5, CSS3, JS
* **Backend:** PHP 8.0
* **Database:** MySQL
* **Environment:** XAMPP<br><br>

## 開發心得與挑戰
這是我人生中第一個獨立開發的全端專案。在短短三個月內，我經歷了從「只懂 HTML/CSS」到「建構完整動態網站」的極速成長。<br><br>

1. 從零到一的全端跨越：

> 挑戰： 一開始對於前後端如何溝通、資料庫如何串接完全沒有概念。

> 克服： 在緊迫的時間壓力下，我採取「邊做邊學」的戰術。雖然過程中曾因對 PHP 邏輯不熟悉而大量依賴 AI 工具協助，但在與 AI 協作除錯的過程中，反而讓我更快速地理解了伺服器端的運作邏輯。

2. 程式架構的體悟與技術債：

> 挑戰：由於初期技術儲備不足且開發時間極度壓縮，我在專案後期才意識到「程式碼模組化」的重要性（例如：將重複功能獨立出來引入，而非重複撰寫）。

> 反思： 雖然目前程式碼仍有冗餘、整潔度有待加強，但這次經驗讓我深刻體會到軟體工程中「DRY (Don't Repeat Yourself)」原則的重要性。這是我在下一個專案中首要改進的目標。

3. 技術選型的抉擇：

> 挑戰： 面對 React 或 Vue 等主流框架的誘惑，我考量到三個月的時程壓力與自身基礎，最終決定選用最熟悉的 HTML/CSS/JS 搭配 PHP 進行開發。

> 收穫： 這讓我打下了扎實的網頁基礎，我明白工具只是手段，能真正解決問題並交付成果才是核心。

4. 意志力的磨練：

> 開發期間，每天早上 10:30 到睡前，我幾乎都在螢幕前度過。除了 coding，更多時間是在研究各國網站的 UI/UX 與配色邏輯。這段「極限開發」的過程雖然辛苦，但看著功能一個個從無到有，帶給我的成就感遠超過身體的疲累。最終獲得了回報：本專案在畢業專題成果展中成功獲得了「優勝」的成績。這不僅是對作品的肯定，更是對我解決問題能力的最佳證明。
